# NextMealAI — Design Context

Descriptive record of the current shipped UI. Follow this, don't invent. Source of truth for `frontend-design` skill.

*Last updated: 2026-04-14 via `/teach-impeccable`.*

## Users

Serious but approachable athletes — know their macros, train consistently, not pros. Geo (AI coach) handles food lookup, plan adjustments, macro math. Primary context: mobile, often one-handed.

## Typography

- **Font:** Inter via `next/font/google`, weights 400/500/600/700
- **Tabular nums** (`tabular-nums`) for all numeric values
- **Section labels:** `text-[10px] uppercase tracking-[0.08em]`
- **Values:** `text-xl` / `text-2xl` with `font-semibold text-accent`

## Radius

`--radius-sm: 8px` · `--radius-md: 12px` · `--radius-lg: 16px` · `--radius-xl: 20px` · `--radius-full: 9999px`. Cards default to `rounded-2xl` (16px). No hard corners.

## Card pattern

```
bg-surface border border-border rounded-2xl p-4
```

Used by `progress-card.tsx`, `quick-stats.tsx`, `next-up-card.tsx`, `macro-progress.tsx` — match this when adding new cards.

## Icons

Lucide React, monochrome, color via text-color classes. Typical size `w-[18px] h-[18px]` on stat cards, larger for section icons.

## Motion

- Framer Motion installed but used sparingly
- CSS transitions for micro-feedback: `transition-colors`, `transition-transform`, `active:scale-[0.97]`
- No bouncy / elastic easing
- Animate `transform` and `opacity` only — never `height`, `width`, `padding`, `margin`
- Respect `prefers-reduced-motion`

## Spacing & layout

- `space-y-3` / `space-y-5` between sections
- Card padding: `p-4` default, `p-5` for hero cards
- Mobile-first, tap targets ≥ 44×44px

## Stack

Next.js 16 App Router · Tailwind v4 · shadcn/ui + Radix · Zustand · React Query v5 · React Hook Form + Zod · Framer Motion · Lucide React.

## Rules

1. **Don't change the palette** — use the tokens above. Don't add new accent colors without asking.
2. **Don't break card pattern** — `bg-surface border border-border rounded-2xl` is the house style.
3. **Don't add motion where there is none** — match what's around the change.
4. **Don't swap fonts** — Inter stays.
5. **Follow what's already there** — sample the nearest existing component and match its patterns before inventing new ones.
